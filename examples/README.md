# Example Data Files

This directory contains example JSON files showing the structure of runtime data files generated by RedHouse.

## Files

### heating_program_schedule_2024-11-06.json
Daily heating program with optimized schedules for geothermal pump control.
- Generated by: `generate_heating_program_v2.py`
- Location: `YYYY-MM/heating_program_schedule_YYYY-MM-DD.json`
- Contains: Load schedules, EVU-OFF periods, cost estimates, temperature parameters

### pump_state.json
EVU cycle tracking state for geothermal pump controller.
- Generated by: `src/control/pump_controller.py`
- Location: `data/pump_state.json`
- Contains: Accumulated ON time, last command, EVU cycle timestamps

### temperature_status.json
Last reading from 1-wire temperature sensors.
- Generated by: `src/data_collection/temperature.py`
- Location: `data_logs/temperature/temperature_YYYY-MM-DD.json`
- Contains: Sensor IDs with temperatures and update timestamps

### spot_price_getter_status.json
Status tracking for electricity spot price collection.
- Generated by: `src/data_collection/spot_prices.py`
- Location: `spot_price_getter_status.json`
- Contains: Latest uploaded price epoch timestamp

### spot_prices_cache.json
Backup cache of raw spot price API response.
- Generated by: `src/data_collection/spot_prices.py`
- Location: `spot_prices_cache.json`
- Contains: Raw hourly electricity prices from spot-hinta.fi API

## Usage

These example files can be used for:
- Understanding data structure/schema
- Testing tools and scripts offline
- Development without InfluxDB access
- Validating schema changes
